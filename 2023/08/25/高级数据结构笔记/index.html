Aggregate Method/聚合法

![[Pasted image 20240228100822.png]]
Remark.
即证明对于一列n个操作，其上界为T(n)时，均摊后得到均摊的复杂度为T(n)/n

Accounting Method/核算法

Remark.
赋予每个操作不同的摊还代价，高于其实际代价的存入信用部分，低于实际代价的则从信用支取，若能保证信用部分总是非负的，则总摊还代价是实际代价的上界

Potential Method/势能法

![[Pasted image 20240228103322.png]]

Remark.
考虑势能函数$\Phi$ ，将数据结构的状态映射到$\mathbb{R}$ ，同时需要保证任意状态下势能均不小于初始状态，以此确保总摊还代价是总实际代价的一个上界

Lec.2 Red-Black tree/B+ tree

Red-Black tree

定义

  1. 节点均为红色或黑色
  2. 根节点和Null节点是黑色的
  3. 红色节点的子节点是黑色的
  4. 对于任意一个节点来说，它到任一子节点的路径上，黑节点数量相同（黑高不包括节点本身）

Lemma
(The height of a red-black tree)
A red-black tree with N internal nodes has the height at most $2ln(N+1)$

Insertion

Always try recoloring first, then go for rotation
![[Pasted image 20240308164504.png]]
Algorithm

  1. Perform standard BST insertion, and color the new node red
  2. if x is the root, color x black, done
  3. if x is not the root, and it’s parent is red, do the following
    a) If x’s uncle is red (then only coloring is enough)
    i) change x’s Uncle and Father to black
    ii) change x’s Grandfather to red
    iii) consider x’s grandfather as new x, repeat (i) and (ii), until done, note that there may accur step 2
    b) If x’s uncle is black
    4 cases, just similar to AVL
    i) Left-left case
    Right rotation of x’s grandfather G,
    G,P color changed
    ii)Left-right case
    Left rotation of x’s parent P,
    then LL case
    G, x color changed
    iii)Right-right case
    iv) Right-left case

Deletion

Since deletion is recursively, so only one-child case and no-child case need to be handled
Note v is the node to be deleted, u is v’s child.
First, perform standard BST deletion, until v has one child or none

If either u or v is red
Simply color the replaced child black

Else
Replace v by u, and color u as double black,
then the task reduces to make double black to single black

Do following(until u is not double black or u is the root)
If u is the root
simply make u single black
Else
note u’s sibling as s
a) s is black and s has at least 1 red child(note the red child as r)
i)RR case
which means that s is the right child of it’s parent, and r is right child of s or both children of s are red
Then, Left rotate s, and change r to black
ii)RL case
which means, s is right child of its parent and r is left child of s
Then, right rotate r, change r and s
iii) LL case
iv) LR case
b) s is black and s has 2 black child
color s red
i)u’s parent p red
color p black, done
ii)u’s parent p black
color p double black, recur for p to remove the double black from it
c) s is red
rotate s to p, change s, p’s color, and it leads to case a or b
